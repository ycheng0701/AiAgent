# AiAgent 核心原子能力迭代路径

> 版本：v0.1  
> 日期：2026-06-18

## 1. 能力分层

Agent 系统可以拆为四层：

| 层级 | 关注点 | 典型问题 |
| --- | --- | --- |
| Model Layer | 推理、生成、结构化输出、多模态理解 | 模型是否能理解任务和生成可靠决策？ |
| Agent Runtime | 循环执行、工具调用、状态、协作、恢复 | 任务能否稳定跑完，失败能否恢复？ |
| Tool / Context Layer | 工具、数据、文件、浏览器、数据库、API、MCP | Agent 是否能安全访问真实环境？ |
| Governance Layer | 权限、安全、评估、观测、成本、人类审批 | 系统是否可上线、可审计、可控？ |

## 2. 原子能力成熟度

### 2.1 感知输入

- L0：只接收用户自然语言。
- L1：能读取文件、网页、数据库片段。
- L2：能识别任务类型、约束、依赖和风险。
- L3：能持续维护任务上下文包，并在上下文过大时压缩。

验收指标：

- 任务意图识别准确率。
- 关键约束遗漏率。
- 上下文引用准确率。

### 2.2 任务规划

- L0：直接回答，无显式计划。
- L1：生成步骤列表。
- L2：区分可并行、需审批、需验证的步骤。
- L3：形成 DAG / 状态机，支持暂停、恢复、重排。

验收指标：

- 计划可执行率。
- 步骤依赖错误率。
- 计划变更后的恢复成功率。

### 2.3 工具调用

- L0：手写少量函数调用。
- L1：工具 schema 化，参数自动校验。
- L2：支持工具注册、发现、权限、重试和结果归一化。
- L3：通过 MCP / 插件市场接入外部工具生态。

验收指标：

- 工具选择准确率。
- 参数校验失败率。
- 工具失败后的恢复率。

### 2.4 记忆管理

- L0：只依赖当前 prompt。
- L1：保留会话历史。
- L2：区分短期记忆、长期记忆、项目知识和用户偏好。
- L3：有写入策略、遗忘策略、冲突检测和记忆质量评估。

验收指标：

- 记忆命中率。
- 错误记忆污染率。
- 引入记忆后的任务成功率变化。

### 2.5 执行控制

- L0：一次模型调用。
- L1：固定 agent loop。
- L2：支持 checkpoint、resume、timeout、retry、idempotency。
- L3：支持沙箱运行、远程执行和多任务调度。

验收指标：

- 长任务完成率。
- 失败恢复成功率。
- 平均执行成本和延迟。

### 2.6 多 Agent 协作

- L0：单 Agent。
- L1：静态角色拆分。
- L2：专家 Agent 作为工具，支持 handoff。
- L3：支持并行协作、冲突仲裁、统一 trace 和最终责任归属。

验收指标：

- 子任务分配准确率。
- 合并结果一致性。
- 多 Agent 相比单 Agent 的质量收益。

### 2.7 人机协同

- L0：最终结果一次性输出。
- L1：危险操作前确认。
- L2：计划、工具调用、关键中间产物可审阅。
- L3：支持人工接管、编辑、回滚和审计。

验收指标：

- 审批拦截有效率。
- 人工介入后的任务恢复率。
- 用户纠错成本。

### 2.8 评估反馈

- L0：人工主观判断。
- L1：保存成功/失败样例。
- L2：建立 golden tasks、自动回归、轨迹评分。
- L3：线上监控和离线评测闭环驱动版本发布。

验收指标：

- 任务成功率。
- 回归发现率。
- 评估与人工判断一致性。

### 2.9 安全治理

- L0：无权限边界。
- L1：敏感工具加确认。
- L2：权限最小化、沙箱、密钥隔离、审计日志。
- L3：针对 prompt injection、tool poisoning、越权访问和供应链风险建立体系化防护。

验收指标：

- 越权调用拦截率。
- 敏感信息泄露率。
- 审计可追溯率。

### 2.10 观测运维

- L0：只看最终输出。
- L1：记录输入输出和工具调用。
- L2：trace/span、成本、延迟、错误分类。
- L3：支持质量、成本、安全、体验四类指标的持续看板。

验收指标：

- 问题定位时间。
- trace 覆盖率。
- 成本异常发现率。

## 3. 推荐建设节奏

| 阶段 | 周期 | 目标 | 交付物 |
| --- | --- | --- | --- |
| Phase 0 | 1-2 周 | 跑通最小 Agent 闭环 | 单 Agent demo、工具 schema、基础日志 |
| Phase 1 | 2-4 周 | 提升可靠性 | checkpoint、retry、trace、测试集 |
| Phase 2 | 4-8 周 | 引入上下文和记忆 | memory store、retrieval、context policy |
| Phase 3 | 8-12 周 | 多 Agent 与工作流 | handoff、parallel task、workflow runtime |
| Phase 4 | 持续 | 生产治理 | sandbox、policy、audit、eval dashboard |

## 4. 优先级建议

短期不要过早追求复杂多 Agent。先把单 Agent 的工具调用、状态恢复、trace 和评估做好，否则多 Agent 只会放大不可控性。

优先顺序：

1. 工具 schema 和权限边界。
2. Agent loop 的 checkpoint / retry / resume。
3. Trace 和任务回放。
4. 上下文压缩和记忆策略。
5. Handoff 与专家 Agent。
6. 生产级安全治理。
